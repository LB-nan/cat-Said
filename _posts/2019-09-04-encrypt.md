公司项目小程序因有收货地址功能，需加密，研究记录一下。

### 简单介绍一下用到的两种加密

MD5 信息摘要算法（英语：MD5 Message-Digest Algorithm），MD5对于同样的字符串加密得到的结果是一致的(可以通过碰撞解密)，MD5的加密过程不可逆。

AES 密码学中的高级加密标准（Advanced Encryption Standard，AES），又称Rijndael加密法，对称加密,是美国联邦政府采用的一种区块加密标准。


### 说一下我加密的思路

前后台动态获取某两个约定的值作为MD5的秘钥，一个作为AES的秘钥，一个作为AES的偏移量，然后后台把要使用AES加密的数据加密，把加密的数据传到前端；
前端获取约定的两个值，使用MD5加密，然后把加密出来的值传递给AES作为秘钥和偏移量解密，把解密到的值进行展示。

缺点： 只能阻拦一些低端的抓包截取。

### 实现步骤
MD5和AES的源码在文章最后贴。
因为是微信小程序，所以示例为微信小程序。
微信小程序不支持使用已经发布的插件包，比如NPM，只能把源码放到项目里面使用。

1. 在小程序根目录创建`utils`文件夹，里面存放源码。
2. 把MD5和AES的js源文件放到文件目录里面。
3. 注意需要在源文件最后 `module.exports` 出来，在你要使用的页面`require`引入。
4. 引入
```
  const md51 = require('../../utils/md5_1.js');
  const aes = require('../../utils/aes.js');
```
5. 使用

```
      /*
    md5 生成两个约定值作为AES的秘钥和偏移量
    使用AES进行两次加密
    */
    let code = md51.md5('jxq1444'); // md5 生成的code

    let v = md51.md5('cfsq1444'); // md5 生成的偏移量

    let key = aes.enc.Utf8.parse(code); // 秘钥

    let iv = aes.enc.Utf8.parse(v); // 偏移量

    let pwd = aes.encrypt('你好', key, iv)  // 单次加密

    let original = aes.encrypt(pwd, key, iv) // 二次加密
    console.log(original)
    let mm = aes.decrypt(original, key, iv); // 解密一次

    let word = aes.decrypt(mm, key, iv); // 解密二次

    console.log(word)  // 原文
```

  /**
   * 加密
   * word：原密码
   * key ：key
   * iv  ： iv
   */
  function encrypt(word, key, iv) {
    var srcs = CryptoJS.enc.Utf8.parse(word);
    var encrypted = CryptoJS.AES.encrypt(srcs, key, { iv: iv, mode: CryptoJS.mode.CBC, padding: CryptoJS.pad.Pkcs7 });
    return encrypted.ciphertext.toString().toUpperCase();
  }

  /**
   * 解密
   * word：加密后的密码
   * key ：key
   * iv  ： iv
   */
  function decrypt(word, key, iv) {
    var encryptedHexStr = CryptoJS.enc.Hex.parse(word);
    var srcs = CryptoJS.enc.Base64.stringify(encryptedHexStr);
    var decrypt = CryptoJS.AES.decrypt(srcs, key, { iv: iv, mode: CryptoJS.mode.CBC, padding: CryptoJS.pad.Pkcs7 });
    var decryptedStr = decrypt.toString(CryptoJS.enc.Utf8);
    return decryptedStr.toString();
  }
